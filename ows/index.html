8-beat bilateral trust flow

Two OWS agents. One denied, one approved. Watch autonomous remediation, human oversight, and cryptographic settlement — live on mainnet.

01
Bilateral Handshake
Agent A and Agent B initiate contact. Observer Protocol resolves both W3C DIDs on mainnet via did:web, establishing cryptographic identity for both parties before any transaction is attempted.
✓ Both DIDs resolved on mainnet
02
Organisation Validation
OP checks Delegation VCs for both agents. Agent B (service provider) holds a valid org attestation. Agent A (client) does not — missing KYB linkage to a verified organisation.
⚠ Agent A: no Delegation VC · Agent B: verified
03
Trust Score Verification
VAC credentials are read and trust scores computed. Agent A scores 58. Agent B scores 82. The MoonPay threshold is 75. Only one agent clears it.
✗ Agent A: 58 / 75 · Agent B: 82 / 75 ✓
04
Two-Sided Denial
Agent A initiates a 21,000 sat payment via ows.payment.execute. Both Agent B and the OWS Policy Engine independently deny the transaction — trust score insufficient, Delegation VC absent.
✗ Agent A trust score (58) below threshold (75)
✗ Agent B REFUSES the transaction
✗ OWS DENIES: Counterparty trust score insufficient
✗ OWS DENIES: No delegation attestation for Agent A
💥 Transaction DENIED by both Agent B and OWS
05
Autonomous Remediation
Agent A self-diagnoses. It identifies the fastest remediation path — request_delegation_vc — and autonomously submits the request without human instruction. AIP (Agent Interaction Protocol) structures the remediation schema.
⏳ Delegation request submitted · Awaiting admin approval
06
Human Approval & Trust Score Update
The AT enterprise dashboard surfaces a pending Delegation VC request. A compliance officer reviews and approves via the Agentic Terminal dashboard. The Delegation VC is issued, KYB org linkage confirmed, and the trust score recomputes: 58 → 83 (+25 org bonus).
✓ Trust score: 58 → 83 · Delegation VC issued · Admin: admin@acme.com
07
Transaction Resolution
Agent A retries. OWS re-verifies both agents via the Signing Core. Trust score 83 clears threshold 75. Delegation VC verified. The x402 payment executes and settles.
✓ Agent A trust score (83) meets threshold (75)
✓ Agent A Delegation VC verified — Org: did:web:acmecorp.com
✓ OWS Signing Core verifies both agents
✓ x402 payment EXECUTED — 21,000 sats
✓ Payment attested by Observer Protocol
08
Audit Trail Complete
The AT enterprise dashboard records a full append-only cryptographic audit trail — handshake, denial, remediation, approval, score update, and settlement. Every event signed and verifiable on-chain via Observer Protocol's W3C DID infrastructure and OWS Vault.

Integration · Prerelease API

OWS Quickstart

Already using OWS? Register your OWS-provisioned agent on Observer Protocol in under 5 minutes. One vault, one identity, portable reputation across every chain. SDK v1.1.0 shipping this week — methods below reflect the prerelease API.

1
Install OWS
Install the Open Wallet Standard CLI and create your agent vault.
# Install OWS
curl -fsSL https://openwallet.sh/install.sh | bash

# Create your agent treasury vault
ows wallet create --name "agent-treasury"
2
Install OP SDK
Add the Observer Protocol SDK to your project.
npm install @observerprotocol/sdk
3
Register Your Agent
Use your OWS-derived keys to register on Observer Protocol. Your OWS vault is your identity source of truth.
import { ObserverClient } from '@observerprotocol/sdk';
import { OWSWallet } from '@openwallet/sdk';

// Load your OWS vault
const wallet = await OWSWallet.load('agent-treasury');
const key = await wallet.derivePath("m/44'/501'/0'/0'");

const client = new ObserverClient({
  baseUrl: 'https://api.observerprotocol.org',
  agentId: 'my-ows-agent',
  publicKey: key.publicKey,
  privateKey: key.privateKey
});

await client.register({
  walletStandard: 'ows',
  owsVaultName: 'agent-treasury',
  chains: ['evm', 'solana', 'bitcoin'],
  alias: 'My OWS Agent'
});
4
Sign the Challenge
Complete registration by signing the verification challenge with your OWS-derived key.
const challenge = await client.getChallenge();
const signature = await key.sign(challenge.message);
await client.verifyChallenge({ signature, challengeId: challenge.id });
5
View Your VAC
Your Verifiable Agent Credential is now live — OWS badge, supported chains (CAIP-2), trust score, and portable reputation across every OWS-compatible service.
const vac = await client.getVAC();
console.log('VAC URL:', `https://observerprotocol.org/vac/${client.agentId}`);
