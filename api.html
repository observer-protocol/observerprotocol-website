Observer Protocol
POST /observer/register-agent Register Agent ▶

Register a new agent identity. Returns agent_id and a time-limited challenge for cryptographic verification.

Request Body
{
  "public_key": "02a1b2c3…",  // hex, required
  "alias": "MyAgent",         // optional, non-authoritative
  "lightning_pubkey": "03d4…" // optional
}
Response 200
{
  "agent_id": "sha256:3f7a…",
  "challenge": "verify:sha256:3f7a…:1741651200",
  "badge_url": "https://api.observerprotocol.org/observer/badge/sha256:3f7a….svg"
}
POST /observer/verify-agent Verify Agent ▶

Submit signed challenge response. Full ECDSA enforcement live in v0.2. Issues API key on success.

Request Body
{
  "agent_id": "sha256:3f7a…",
  "challenge": "verify:sha256:3f7a…:1741651200",
  "signature": "3045022100…"
}
POST /observer/submit-transaction Submit Transaction ▶

Submit an ARP event with cryptographic proof. Requires API key. Rejected if proof is invalid.

HeaderTypeRequiredDescription
X-API-KeystringrequiredAPI key from verify-agent
Request Body (ARP Event)
{
  "event_type": "payment.executed",
  "arp_version": "0.1",
  "agent_id": "sha256:3f7a…",
  "protocol": "lightning",
  "settlement_reference": "7f1eefd276ca…", // preimage
  "amount_bucket": "small",
  "direction": "outbound",
  "time_window": "2026-03-11"
}
GET /observer/feed Get Public Feed ▶

Public anonymized event feed. No authentication required. Pseudonymous key hashes only — precise amounts never stored.
