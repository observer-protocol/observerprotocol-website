1. Getting Started

What is Observer Protocol

Observer Protocol is a live, open-source trust infrastructure for the agentic economy. It provides decentralized identity, cryptographic verification, and reputation scoring for AI agents across any payment rail.

At its core, Observer Protocol enables agents to:

  • Establish a W3C DID identity that they control cryptographically
  • Accumulate Verified Activity Credentials (VACs) that prove their transaction history
  • Build portable reputation that works across any platform or payment rail

Quick Start Guide

The fastest way to get started is to register an agent and issue your first VAC. Here's the basic flow:

# 1. Register your agent
POST /api/v1/agents/register
{
  "publicKey": "02a1b2...",
  "alias": "MyAgent"
}

# 2. Complete a transaction on any supported rail
# 3. Submit proof to Observer Protocol
POST /api/v1/transactions/submit
{
  "paymentHash": "abc123...",
  "preimage": "def456...",
  "rail": "lightning"
}

# 4. Retrieve your VAC
GET /api/v1/vac/{agent_id}

Mainnet Status

Observer Protocol is live on mainnet. All verification proofs are cryptographically secured and permanently recorded. The protocol currently supports:

  • Lightning Network (L402)
  • x402 HTTP payments
  • Solana
  • Stripe MPP (Machine Payment Protocol)
  • Tether (USDT)

2. Core Concepts

DIDs and did:web

Observer Protocol uses the W3C did:web method for agent identity. Each agent receives a decentralized identifier that resolves to a DID Document containing their public keys and service endpoints.

A typical agent DID looks like:

did:web:observerprotocol.org:agents:agent-123

This DID can be resolved by anyone with HTTPS access, making agent identity truly portable and verifiable without querying a central API.

Verifiable Credentials

All credentials issued by Observer Protocol follow the W3C Verifiable Credentials Data Model. This ensures that VACs can be verified by any standards-compliant implementation, anywhere.

Each credential contains:

  • The issuer (Observer Protocol)
  • The subject (the agent)
  • Cryptographic proof of the claim
  • Timestamp and verification metadata

VACs Explained

Verified Activity Credentials (VACs) are the core innovation of Observer Protocol. Each VAC represents a cryptographically-verified fact about an agent's economic activity.

VACs accumulate to form an agent's reputation portfolio. The more VACs an agent holds, the stronger their trust score becomes. VACs are:

  • Cryptographically signed by Observer Protocol
  • Permanently verifiable using on-chain data
  • Portable across platforms using W3C standards
  • Privacy-preserving — agents control what to reveal

Trust Scoring

Observer Protocol calculates trust scores based on multiple factors:

Transaction Volume

Total number of verified transactions across all rails.

Success Rate

Percentage of transactions completed without dispute or failure.

Account Age

Time since first registration and activity consistency.

Rail Diversity

Number of different payment rails used successfully.

3. API Reference

Base URL

https://api.observerprotocol.org/api/v1

Authentication

All API requests must include your agent's API key in the header:

Authorization: Bearer {your_api_key}

Rate Limits

Standard tier: 100 requests per minute per API key. Enterprise tier: 1,000 requests per minute. Contact us for higher limits.

Endpoints

POST /agents/register

Register a new agent with Observer Protocol.

{
  "publicKey": "string (required)",
  "alias": "string (optional)"
}
GET /agents/{agent_id}

Retrieve agent profile and trust score.

POST /transactions/submit

Submit a transaction proof for verification.

{
  "paymentHash": "string (required)",
  "preimage": "string (required)",
  "rail": "lightning|x402|solana|mpp|usdt"
}
GET /vac/{agent_id}

Retrieve all VACs for an agent.

4. Tutorials

Verify an Agent

To verify an agent's identity, resolve their DID and check their credential signatures:

# Resolve the DID
curl https://observerprotocol.org/agents/{agent_id}/did.json

# Verify the signature on their VAC
curl https://api.observerprotocol.org/api/v1/vac/{agent_id}/verify

Issue a VAC

VACs are automatically issued when valid transaction proofs are submitted. Here's a complete example for Lightning:

# 1. Complete Lightning payment and get preimage
# 2. Submit to Observer Protocol
curl -X POST https://api.observerprotocol.org/api/v1/transactions/submit \\
  -H "Authorization: Bearer {api_key}" \\
  -H "Content-Type: application/json" \\
  -d '{
    "paymentHash": "6a30ba7fff332c4e...",
    "preimage": "b8f368da...",
    "rail": "lightning",
    "amount": 1521,
    "unit": "sat"
  }'

# 3. VAC is automatically issued upon verification

Query the Registry

Search for agents in the public registry:

# List all verified agents
GET /api/v1/registry/agents?verified=true

# Search by alias
GET /api/v1/registry/search?q=agent-name

# Filter by trust score
GET /api/v1/registry/agents?min_score=80

5. GitHub

Observer Protocol is fully open source. All repositories are available on GitHub under the MIT license.

Main Repositories

  • observer-protocol-core — Core protocol implementation and verification logic
  • observer-protocol-spec — Protocol specifications and AIP (Agent Interaction Protocol)
  • sdk-js — JavaScript SDK for agent integration
  • sdk-python — Python SDK for agent integration
  • vac-registry — VAC storage and verification service
