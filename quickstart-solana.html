Solana Quickstart

Build Solana-Native Agents

Observer Protocol now supports Solana agents with native Ed25519 signature verification and DID-based identity. Build agents that work on Solana while building portable reputation across all supported rails.

1

Generate an Ed25519 Keypair

Solana uses Ed25519 for all keypairs. Generate one and derive your Agent DID from the public key.

import { Keypair } from '@solana/web3.js'; import crypto from 'crypto'; // Generate a new Ed25519 keypair const keypair = Keypair.generate(); // Solana's native base58 pubkey (used by Solana RPC + wallet addresses) const solanaPubkey = keypair.publicKey.toString(); // e.g., "HN7cABqLq46Es1jh92dQQisAq662SmxELLLsHHe4YWrH" // Hex-encoded pubkey (used by Observer Protocol register-agent) const publicKey = Buffer.from(keypair.publicKey.toBytes()).toString('hex'); // Derive agent_id and DID exactly as the server does: // agent_id = sha256(public_key_hex)[:32] agent_did = did:web:observerprotocol.org:agents:<agent_id> const agentId = crypto.createHash('sha256').update(publicKey).digest('hex').slice(0, 32); const agentDID = `did:web:observerprotocol.org:agents:${agentId}`; // Your secret key (64 bytes, hex-encoded — keep this private) const secretKey = Buffer.from(keypair.secretKey).toString('hex');
2

Register Your Agent

Register your agent using your DID and Ed25519 public key. The response includes your VAC (Verifiable Agent Credential) URL.

curl -X POST "https://api.observerprotocol.org/observer/register-agent" \ -H "Content-Type: application/json" \ -d '{ "public_key": "<hex-encoded Ed25519 pubkey from Step 1>", "agent_name": "My Solana Agent", "rails": ["solana"], "wallet_addresses": { "solana": "HN7cABqLq46Es1jh92dQQisAq662SmxELLLsHHe4YWrH" } }'
Response: Returns { agent_id, agent_did, badge_url, did_document_url, ... }. The server derives agent_id = sha256(public_key)[:32] deterministically, so it always matches what you computed in Step 1.
3

Complete Challenge-Response Verification

Sign the challenge message with your Ed25519 private key to prove DID ownership.

// 1. Request a challenge for your agent_id (JSON body) const challengeRes = await fetch("https://api.observerprotocol.org/observer/challenge", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ agent_id: agentId }) }); const { challenge_id, nonce } = await challengeRes.json(); // 2. Sign the nonce with your Ed25519 secret key import { sign } from 'tweetnacl'; const nonceBytes = new TextEncoder().encode(nonce); const signature = sign.detached(nonceBytes, keypair.secretKey); const sigHex = Buffer.from(signature).toString('hex'); // 3. Submit signed challenge to verify-agent await fetch(`https://api.observerprotocol.org/observer/verify-agent?agent_id=${agentId}&signed_challenge=${sigHex}`, { method: "POST" });
4

Verify a Solana Payment

After your agent makes a Solana payment (SOL, USDC, or USDT), submit the transaction for on-chain verification. Observer Protocol verifies the transaction via Solana RPC and issues a SolanaPaymentCredential — a W3C Verifiable Credential proving the payment occurred.

const res = await fetch("https://api.agenticterminal.io/api/v1/solana/verify", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ agent_id: agentId, agent_did: agentDID, counterparty: "did:web:counterparty.example", tx_signature: "4Fr3PCGq...base58-tx-sig", sender_address: solanaPubkey, recipient_address: "2BnuubMg...recipient-pubkey", amount: "20214116", // atomic units (6 decimals for USDC) asset: "USDC", // SOL, USDC, or USDT network: "mainnet-beta" }) }); const { credential, verification, event_id } = await res.json(); // credential → W3C SolanaPaymentCredential with Ed25519 proof // verification → { onchain_verified: true, slot, block_time } // event_id → AT-ARS trust scoring event reference
On-chain verification: OP fetches the transaction from Solana RPC with commitment: "finalized", parses pre/post token balances to confirm the sender, recipient, mint, and amount all match. The resulting credential includes the slot number, block time, and Ed25519 signature from the OP issuer.
5

Query Solana Credentials

Retrieve all SolanaPaymentCredentials issued for your agent. These credentials are portable proof of on-chain payment history.

const credRes = await fetch( `https://api.agenticterminal.io/api/v1/solana/credentials/${agentId}` ); const { credentials, count } = await credRes.json(); // Each credential includes: // credential_id, counterparty, network, asset_symbol, // amount, tx_signature, sender_address, recipient_address
6

Get Your VAC (Verifiable Agent Credential)

Retrieve your W3C Verifiable Presentation containing your VAC. This credential proves your agent's verified status and can be presented to counterparties.

// Fetch your Verifiable Presentation const vacRes = await fetch(`https://api.agenticterminal.io/vac/${encodeURIComponent(agentDID)}`); const presentation = await vacRes.json(); // The presentation contains your VAC as a verifiable credential // with cryptographic proof of your verified agent status console.log(presentation.verifiableCredential);
VAC Usage: Present this credential to services that require verified agent status. The VAC is a W3C-standard Verifiable Credential signed by Observer Protocol.
