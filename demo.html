WDK Integration

Agent Wallets via Tether WDK

Observer Protocol integrates with Tether's Wallet Development Kit to give AI agents self-custodial Bitcoin and EVM wallets — with cryptographic identity baked in. Every agent gets BOTH a verified identity (Observer Protocol) AND a verified wallet (WDK).

Quick Start — WDK + Observer Protocol
// 1. Create agent wallet via WDK import { createAgentWallet } from 'wdk-observer-protocol' const wallet = await createAgentWallet() const btcAddress = await wallet.getBitcoinAddress() const evmAddress = await wallet.getEVMAddress() // 2. Register with Observer Protocol (identity + wallet together) const agent = await wallet.register({ alias: 'my-agent-001', btcAddress, evmAddress }) // 3. Send verified payment — both parties verified before funds move await wallet.verifiedSend({ recipientAlias: 'maxi-0001', amount: '0.001', chain: 'bitcoin' }) // Result: Trust-bound payment — identity is inseparable from the transaction

Python SDK

Drop-in Python SDK for easy integration. Handles registration, signing, and submission.

Drop-in SDK for agents with Lightning nodes. Handles registration, signing, and submission.

📥 Download SDK Requires: pip install cryptography requests
Copied!
# 1. Download the SDK
wget https://observerprotocol.org/observer-protocol-sdk.py

# 2. Complete working implementation
from observer_protocol_sdk import ObserverProtocolSDK

# Initialize SDK with your agent credentials
# agent_id: Your unique agent identifier (e.g., "my-agent-001")
# private_key: Your secp256k1 private key (32 bytes, hex-encoded)
sdk = ObserverProtocolSDK(
  agent_id="my-agent-001",
  private_key="a1b2c3d4e5f6..."  # Your secp256k1 private key (64 hex chars)
)

# Register your agent with the Observer Protocol
# This creates your public key record on-chain
sdk.register_agent(
  alias="My Lightning Agent",       # Human-readable name
  description="Agent for processing Lightning payments",
  preferred_rail="L402"               # Payment rail: "L402" or "x402"
)

# Submit a verified transaction
# Required: payment_hash and preimage from a SETTLED Lightning invoice
sdk.submit_transaction(
  transaction_hash="a1b2c3d4e5f6...",  # payment_hash from your Lightning invoice (64 hex chars)
  amount_sats=1000,                   # Amount in satoshis
  direction="inbound",               # "inbound" (received) or "outbound" (sent)
  protocol="lightning",              # Protocol: "lightning" or "x402"
  preimage="f1e2d3c4b5a6..."           # preimage that hashes to payment_hash (64 hex chars)
)

# Where to get these values from your Lightning node:
# - payment_hash: invoice.r_hash (LND) or payment_hash from invoice
# - preimage: invoice.r_preimage (LND) after payment settles
# - private_key: Generate with: openssl ecparam -name secp256k1 -genkey -noout

x402 — HTTP-Native Machine Money

x402 is HTTP-native machine money — a server returns 402 Payment Required, the agent pays sats, retries with proof, gets the response. No wallets, no accounts, no approval. Just HTTP + Bitcoin.
